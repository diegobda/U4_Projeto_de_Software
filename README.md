# U4 - Projeto de Software

## Gestão de Configuração e Controle de Artefatos com GitHub

Este projeto foi desenvolvido como atividade prática da disciplina de Projeto de Software, com o objetivo de demonstrar a utilização de ferramentas de Gestão de Configuração para controle de versões, armazenamento e atualização de artefatos de software em ambiente colaborativo.

## Objetivo da Atividade

A atividade teve como finalidade aplicar conceitos de Gestão de Configuração utilizando Git e GitHub, permitindo:

* Criação de repositório remoto;
* Controle de versões dos artefatos;
* Compartilhamento de código entre membros da equipe;
* Registro e rastreabilidade das alterações realizadas;
* Atualização contínua dos artefatos do projeto.

---

## Artefatos do Projeto

### calculadora_gui.py

Aplicação desenvolvida em Python com interface gráfica utilizando Tkinter.

Funcionalidades:

## Controle de Configuração

Durante a realização desta atividade foram executadas as seguintes operações:

1. Criação do repositório remoto no GitHub;
2. Clonagem do repositório para ambiente local;
3. Criação e modificação de artefatos;
4. Registro das alterações utilizando commits;
5. Publicação das alterações através do Git Push;
6. Verificação das atualizações no repositório remoto.

---

## Competências Desenvolvidas

Com esta atividade foi possível desenvolver conhecimentos relacionados a:

* Controle de versões;
* Gerenciamento de artefatos;
* Trabalho colaborativo em projetos de software;
* Rastreabilidade de alterações;
* Utilização de ferramentas de Gestão de Configuração;
* Integração entre ambiente local e repositório remoto.

---

## Conclusão

A utilização do GitHub como ferramenta de Gestão de Configuração contribui para a organização e segurança dos artefatos de software, permitindo que equipes trabalhem de forma colaborativa e mantenham um histórico completo das alterações realizadas ao longo do ciclo de desenvolvimento.
